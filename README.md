# Urban Routes - Test Design

Projeto de design de testes para o formulário “Adicionar carteira de motorista” do Urban Routes, com classes de equivalência, valores-limite e casos de teste.

## Sobre o projeto
Este repositório apresenta uma atividade de QA focada em design de testes com base nos requisitos do recurso de compartilhamento de carros do Urban Routes.

O objetivo foi analisar os campos do formulário “Adicionar carteira de motorista” e preparar a documentação necessária para a futura execução dos testes.

## Objetivo
Aplicar técnicas de design de testes para identificar classes de equivalência, valores-limite e elaborar casos de teste com base nos requisitos do sistema.

## Escopo analisado
- Campo "Nome"
- Campo "Sobrenome"
- Campo "Data de nascimento"

## Técnicas utilizadas
- Classes de equivalência
- Análise de valores-limite
- Escrita de casos de teste

## Arquivos deste repositório
- `equivalence-classes.md` → classes de equivalência dos campos analisados
- `boundary-values.md` → valores-limite identificados
- `test-cases-name-field.md` → casos de teste do campo "Nome"

## Competências demonstradas
- Design de testes
- Interpretação de requisitos
- Classes de equivalência
- Valores-limite
- Escrita de casos de teste
- Atenção aos detalhes
- Organização da documentação

## Observação
Este projeto foi desenvolvido como atividade prática de formação em QA e tem foco em preparação da documentação de testes, sem execução do sistema nesta etapa.
